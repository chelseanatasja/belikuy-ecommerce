2. Spesifikasi dan Fitur Sistem
1. Arsitektur Sistem
Sistem e-commerce bertipe Marketplace ini menggunakan model Client-Server dengan pembagian hak akses (Role-Based Access).
* Frontend (UI Gateway): Dibangun dengan Streamlit (Python). Sistem akan memiliki 3 antarmuka/portal berbeda:
   1. Storefront (Untuk Pembeli)
   2. Seller Center (Untuk Penjual/Vendor mengelola tokonya)
   3. Super Admin Dashboard (Untuk pemilik aplikasi BeliKuy! memantau seluruh aktivitas).
* Backend: Node.js bertugas menangani logika bisnis (API), memisahkan request antara pembeli, penjual, dan sistem pembayaran.
* Database: Relasional SQL (MySQL/PostgreSQL) terpusat. Tabel akan sangat bergantung pada Foreign Key untuk menghubungkan entitas Users, Stores, Products, dan Orders.

3. Spesifikasi Fungsional (Fitur Utama)
1. Modul Pembeli (Storefront)
* Pencarian Global & Filter: Mencari produk dari berbagai toko dengan filter kategori, harga, dan rating.
* Keranjang Belanja Pintar: Manajemen item belanja yang otomatis dikelompokkan berdasarkan toko penjual (multi-store cart).
* Checkout & Pembayaran: Integrasi sistem pembayaran (Transfer/E-wallet) dan pemilihan kurir.
* Riwayat Transaksi (Order History): Halaman khusus bagi pembeli untuk melihat daftar pesanan yang sedang berjalan maupun yang sudah selesai.
2. Modul Penjual (Seller Center)
* Manajemen Toko: Fitur untuk membuka toko, mengatur profil, banner, dan deskripsi toko.
* Manajemen Produk Mandiri (CRUD): Hak akses penuh untuk menambah, menghapus, atau memperbarui detail produk dan stok milik tokonya sendiri.
* Laporan Pendapatan Toko: Dashboard sederhana bagi penjual untuk memantau total omzet dan saldo yang bisa ditarik.
3. Modul Pemrosesan Pesanan & Tracking (Order Service)
* Penerimaan & Routing Pesanan: Menangkap data transaksi masuk dan memecahnya ke masing-masing toko jika pembeli membeli dari beberapa vendor sekaligus.
* Manajemen Status Pengiriman: Fitur bagi penjual untuk memperbarui status (Pending, Packed, Shipped, Delivered) yang dapat dilacak secara real-time oleh pembeli.
* Kalkulasi Biaya & Ongkir: Sistem penghitungan otomatis untuk total belanja, pajak aplikasi, dan biaya pengiriman berdasarkan lokasi toko.
4. Modul Super Admin (Owner Dashboard)
* Manajemen Vendor: Verifikasi pendaftaran toko baru dan otoritas untuk memblokir toko yang melanggar aturan.
* Pantauan Transaksi Global: Monitoring seluruh aliran uang dan aktivitas transaksi yang terjadi di seluruh platform BeliKuy!.
* Laporan Pendapatan Platform: Analitik terkait komisi yang didapat penyedia platform dari setiap transaksi sukses.
5. Modul Manajemen Pengguna & Keamanan (User & Auth Service)
* Registrasi & Login (Authentication): Sistem pendaftaran dan masuk akun yang aman (bisa menggunakan JWT untuk backend Node.js).
* Manajemen Hak Akses (RBAC): Proteksi halaman agar user hanya bisa mengakses fitur sesuai perannya (Pembeli, Penjual, atau Admin).
* Manajemen Profil & Alamat: Penyimpanan data diri, buku alamat pengiriman (untuk pembeli), dan data rekening bank (untuk penjual).
* Manajemen Sesi (Session Management): Integrasi st.session_state di Streamlit agar status login tetap terjaga selama pengguna berpindah-pindah halaman.


4. Spesifikasi Perangkat Keras (Hardware)
Untuk mendukung kelancaran operasional dan pengembangan platform e-commerce BeliKuy ("Lapak Luas, Belanja Puas"), berikut adalah spesifikasi perangkat keras minimum yang dibutuhkan:
1. Server Lokal (Development & Database Hosting)
Lingkungan server lokal ini digunakan untuk menjalankan script web (seperti Streamlit) dan memproses query relasional pada database (seperti eksekusi relasi tabel antar vendor dan product).
* Processor: Intel Core i3 Generasi ke-10 (atau prosesor AMD yang setara).
* RAM: 8 GB (Kapasitas minimal ini sangat disarankan agar server web lokal dan database engine SQL dapat berjalan bersamaan tanpa bottleneck).
* Storage: 256 GB SSD (Penggunaan Solid State Drive diwajibkan untuk menjamin kecepatan baca/tulis data, terutama saat melakukan validasi query database dengan banyak relasi tabel).
* Operating System: Windows 10/11 atau Ubuntu Server.
2. Perangkat Akses Operasional (Super Admin & Vendor) 
Mengingat BeliKuy menggunakan konsep multi-company, dashboard operasional akan diakses oleh dua pihak utama: pengelola platform (Super Admin) dan para penjual (Vendor).
* PC/Laptop: Perangkat dengan minimum RAM 4 GB sudah memadai untuk mengakses dashboard analitik dan manajemen produk melalui web browser.
* Perangkat Cetak: Printer thermal atau printer label pengiriman. Perangkat ini secara khusus dibutuhkan oleh pihak Vendor untuk mencetak resi pengiriman dari pesanan pelanggan secara efisien.


3. Database
-- =====================================
-- DATABASE BELIKUY

CREATE DATABASE IF NOT EXISTS belikuy;
USE belikuy;


-- =====================================
-- 1. USERS (MASTER)
-- =====================================
CREATE TABLE Users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    role ENUM('admin', 'customer', 'seller') DEFAULT 'customer',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


-- =====================================
-- 2. COMPANIES / SELLER (MASTER)
-- =====================================
CREATE TABLE Companies (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    company_name VARCHAR(100) NOT NULL,
    address TEXT,
    rating DECIMAL(2,1) DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES Users(id) ON DELETE CASCADE
);


-- =====================================
-- 3. CATEGORIES (MASTER)
-- =====================================
CREATE TABLE Categories (
    id INT AUTO_INCREMENT PRIMARY KEY,
    category_name VARCHAR(100) NOT NULL
);


-- =====================================
-- 4. SUPPLY COMPANIES (MASTER)
-- =====================================
CREATE TABLE Supply_Companies (
    id INT AUTO_INCREMENT PRIMARY KEY,
    supply_company_name VARCHAR(100) NOT NULL,
    contact_number VARCHAR(20),
    address TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


-- =====================================
-- 5. PRODUCTS (MASTER)
-- =====================================
CREATE TABLE Products (
    id INT AUTO_INCREMENT PRIMARY KEY,
    company_id INT,
    category_id INT,
    supply_company_id INT,
    brand VARCHAR(100) NOT NULL,
    product_name VARCHAR(150) NOT NULL,
    size VARCHAR(50) DEFAULT NULL,
    color VARCHAR(50) DEFAULT NULL,
    description TEXT,
    price DECIMAL(10, 2) NOT NULL,
    stock INT NOT NULL DEFAULT 0,
    image_url VARCHAR(255), 
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,


    FOREIGN KEY (company_id) REFERENCES Companies(id) ON DELETE CASCADE,
    FOREIGN KEY (category_id) REFERENCES Categories(id) ON DELETE SET NULL,
    FOREIGN KEY (supply_company_id) REFERENCES Supply_Companies(id) ON DELETE SET NULL
);


-- =====================================
-- 6. USER ADDRESSES (MASTER)
-- =====================================
CREATE TABLE User_Addresses (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    address TEXT NOT NULL,
    city VARCHAR(100),
    postal_code VARCHAR(10),
    is_default BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (user_id) REFERENCES Users(id) ON DELETE CASCADE
);


-- =====================================
-- 7. SHIPMENT COMPANIES (MASTER)
-- =====================================
CREATE TABLE Shipment_Companies (
    id INT AUTO_INCREMENT PRIMARY KEY,
    company_name VARCHAR(50) NOT NULL,
    service_type VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


-- =====================================
-- 8. ORDERS (TRANSACTION)
-- =====================================
CREATE TABLE Orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    address_id INT,
    total_price DECIMAL(10, 2) NOT NULL,
    status ENUM('pending', 'paid', 'shipped', 'completed', 'cancelled') DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,


    FOREIGN KEY (user_id) REFERENCES Users(id) ON DELETE CASCADE,
    FOREIGN KEY (address_id) REFERENCES User_Addresses(id)
);


-- =====================================
-- 9. ORDER ITEMS (TRANSACTION)
-- =====================================
CREATE TABLE Order_Items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_id INT,
    product_id INT,
    quantity INT NOT NULL,
    price DECIMAL(10,2),
    subtotal DECIMAL(10, 2) NOT NULL,


    FOREIGN KEY (order_id) REFERENCES Orders(id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES Products(id) ON DELETE CASCADE
);


-- =====================================
-- 10. PAYMENTS (TRANSACTION)
-- =====================================
CREATE TABLE Payments (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_id INT,
    payment_method ENUM('transfer', 'ewallet', 'cod'),
    payment_status ENUM('pending', 'paid', 'failed') DEFAULT 'pending',
    paid_at TIMESTAMP NULL,


    FOREIGN KEY (order_id) REFERENCES Orders(id) ON DELETE CASCADE
);


-- =====================================
-- 11. SHIPMENTS (TRANSACTION)
-- =====================================
CREATE TABLE Shipments (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_id INT,
    company_id INT,
    shipment_company_id INT,
    tracking_number VARCHAR(100),
    shipping_status ENUM('pending', 'shipped', 'delivered') DEFAULT 'pending',


    FOREIGN KEY (order_id) REFERENCES Orders(id) ON DELETE CASCADE,
    FOREIGN KEY (company_id) REFERENCES Companies(id) ON DELETE CASCADE,
    FOREIGN KEY (shipment_company_id) REFERENCES Shipment_Companies(id) ON DELETE SET NULL
);


-- =====================================
-- 12. REVIEWS (TRANSACTION)
-- =====================================
CREATE TABLE Reviews (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    product_id INT,
    rating INT CHECK (rating BETWEEN 1 AND 5),
    comment TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,


    FOREIGN KEY (user_id) REFERENCES Users(id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES Products(id) ON DELETE CASCADE
);

               4. UI & UX Design
               1. User Experience
               1. Pendekatan Utama (UX Philosophy)
Pendekatan User Experience pada aplikasi BeliKuy berpusat pada konsep "Aesthetic Usability Effect", di mana desain visual yang menarik (menggunakan perpaduan soft pink dan soft purple) dimanfaatkan untuk meningkatkan kenyamanan dan toleransi pengguna, tanpa mengorbankan fungsionalitas. Aplikasi dirancang dengan prinsip classy, clean, dan simple guna meminimalisir beban kognitif (cognitive load) pengguna saat menavigasi aplikasi, sehingga proses belanja maupun pengelolaan toko menjadi intuitif.


               2. Arsitektur Informasi & Navigasi Tersegmentasi (Role-Based)
BeliKuy menerapkan arsitektur informasi yang dinamis berdasarkan Role-Based Access Control (RBAC) untuk menjaga fokus masing-masing pengguna: 
               * Store Front (Pembeli): Menggunakan pola navigasi eksploratori dengan Top Navbar. Pendekatan ini membebaskan ruang layar vertikal untuk menampilkan grid produk secara maksimal (visual-heavy). Search bar diletakkan secara prominen untuk memfasilitasi goal-oriented users yang ingin mencari barang secara spesifik dan cepat.
               * Seller Center & Super Admin: Menggunakan pendekatan task-oriented dengan Sidebar Navigation. Sidebar memberikan struktur hierarki yang stabil dan permanen, memudahkan penjual dan admin untuk berpindah antar modul operasional (seperti manajemen produk, pesanan, dan laporan) secara efisien tanpa kehilangan konteks halaman.


















               3. Pemetaan User Journey & Interaksi
Setiap alur pengguna dirancang untuk meminimalisir gesekan (frictionless experience): 
               * Fase Discovery & Keputusan (Pembeli): Pengguna disajikan informasi yang mudah dipindai (scannable) melalui card-based design dengan soft shadows. Pada halaman detail produk, hierarki visual diarahkan secara tegas pada gambar produk, harga, dan tombol CTA (Call-to-Action) "Add to Cart" yang kontras.
               * Fase Checkout (Multi-store Cart): Mengingat sifatnya sebagai marketplace, UX pada keranjang belanja dikelompokkan berdasarkan toko. Subtotal per toko disajikan secara transparan untuk mencegah kebingungan pengguna, dilanjutkan dengan alur checkout linear (Alamat -> Kurir -> Pembayaran) yang terstruktur dan terprediksi.
               * Fase Manajemen & Operasional (Penjual & Admin): Dashboard dirancang menggunakan visualisasi data yang sederhana (grafik dasar dan statistik numerik) agar pengguna dapat menyerap ringkasan performa toko/platform hanya dengan sekilas pandang (glanceable insights). Proses CRUD (Create, Read, Update, Delete) pada produk dan manajemen order disajikan dalam format tabel/grid yang responsif.


               4. Hierarki Visual dan Readability
Untuk memastikan kenyamanan penggunaan dalam durasi yang lama, tipografi dirancang secara matang. Penggunaan Plus Jakarta Sans pada heading memberikan karakter modern-playful yang sesuai dengan identitas brand, sedangkan Inter pada body text menjamin tingkat keterbacaan (readability) yang tinggi pada ukuran layar yang kecil. CTA selalu diutamakan dengan warna gradient accent agar selalu menonjol dari latar belakang neutral/off-white. 


               5. Micro-interactions & Umpan Balik (Feedback System)
Untuk membuat sistem terasa "hidup" dan responsif, aplikasi mengintegrasikan micro-interactions berupa animasi hover yang ringan pada product card dan transisi smooth pada modal atau dropdown. Interaksi ini memberikan umpan balik instan setiap kali pengguna melakukan aksi, sekaligus mendukung elemen kawaii minimalis yang memperkuat identitas emosional aplikasi. 


               6. Aksesibilitas & Responsivitas 
Seluruh layout mengadopsi prinsip airy layout (spasi yang longgar) dan rounded corners yang ramah di mata. Desain juga dipastikan mobile-friendly, di mana komponen-komponen utama dirancang adaptif terhadap berbagai ukuran layar (responsive design), memastikan pengalaman yang konsisten baik saat diakses melalui desktop maupun perangkat mobile.
